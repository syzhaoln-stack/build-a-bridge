# 全书编辑方案

## 定位

《请把手弄脏》是一部由桥引发的教学散文。它不按桥型讲授桥梁知识，也不罗列 AI 工具，而是让桥梁、AI、教学与应用在真实课堂和项目现场中彼此照亮。

主要读者是高校工科教师、研究生助教、专业负责人和教师发展工作者。读者不必懂桥梁，但应能在读后重做一个 45--90 分钟教学单元，使学生留下作品、修改记录、判断依据和 AI 使用证据。

## 五部分十五章

| 部分 | 思考姿态 | 三章的推进 |
|---|---|---|
| 梁桥 | 最直接地跨越问题 | AI 与人的学习；为什么愿意再试；“有什么用” |
| 拱桥 | 把压力转化为结构 | AI 答案进入课堂；课堂微型制度；美丽的初稿 |
| 斜拉桥 | 让反馈直接传力 | 先给魅力；理论进入手感；让学习发生 |
| 悬索桥 | 用深锚支持远跨 | 判断权；教师反馈；责任与专业底线 |
| 组合桥 | 按条件组合方法 | TRIZ 作为启发式；邀请参与；回到行动 |

## 单章写法

每章优先保持散文的流动，不套统一论文模板，但至少应完成四件事：

1. 从一个可定位的录音、项目动作或作者原文进入；
2. 让桥梁意象推动问题，而不是为了对应桥型硬讲知识；
3. 把作者判断、课堂观察和外部研究分开；
4. 结尾给读者一个可以带回课堂的动作。

目标长度为每章约 6--10 个网页屏，不以 PDF 页数作为当前完成标准。

## 真实性边界

- 不补写学生对白、次数、神态和结果；
- 课堂口语中的数字保留其语境，不改写成普遍规律；
- 原型和设想不写成已经验证的学习效果；
- “AI 永远不会”改为当前责任边界或需要继续追问的问题；
- 学生作品公开前需匿名化并取得授权。

## 当前交付

GitHub Pages 是主要阅读界面：电脑端固定侧边目录，手机端折叠菜单，十五章独立页面。LaTeX 保留为可选排版来源，但暂不以 PDF 为优先交付。
