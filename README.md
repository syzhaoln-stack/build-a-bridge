# 请把手弄脏：AI时代的学与玩，或一座桥的哲学

> **120万字课堂录音，2年教学实验，1个核心问题：当机器能比人更快地学会一切，人为什么还要学习？**

这不是一本"AI工具使用指南"。这是一位工科教师用桥梁思维重新设计课堂的哲学记录。

## 在线体验

[SpanCraft 桥造计划](https://syzhaoln-stack.github.io/spancraft-bridge-game/) —— 书中提到的桥梁建造游戏，你可以在线搭一座桥。

## 目录

全书用五种桥型作为组织隐喻：

| 桥型 | 隐喻 | 章节 | 核心问题 | 状态 |
|------|------|------|---------|------|
| 🌉 **梁桥** | 最朴素的跨越 | 第1–3章 | AI如何学习，人如何学习？ | ✅ 完成 |
| 🏛️ **拱桥** | 压力创造强度 | 第4–6章 | 新范式如何撞击旧系统？ | 📝 待写 |
| 🎻 **斜拉桥** | 每一根索都直接传递力 | 第7–9章 | 学习回路如何设计？ | 📝 待写 |
| ⛓️ **悬索桥** | 跨越最长距离，依赖最深的锚 | 第10–12章 | 什么不能外包给AI？ | 📝 待写 |
| 🔀 **组合桥** | 没有一种桥型能解决所有问题 | 第13–15章 | 如何邀请所有人加入？ | 📝 待写 |
